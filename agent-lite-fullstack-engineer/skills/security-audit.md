# skills/security-audit.md — 安全检查清单

> 上线前花5分钟检查，避免90%的安全问题。

---

## ✅ 上线前必查

```markdown
## 代码检查
- [ ] 密码用 bcrypt 哈希（不用 MD5/SHA1）
- [ ] SQL 用 ORM 或参数化查询
- [ ] 用户输入有验证（Zod/Joi）
- [ ] 敏感信息不硬编码
- [ ] API 有错误处理

## 配置检查
- [ ] 环境变量已设置（.env 不提交）
- [ ] HTTPS 已启用
- [ ] 数据库密码够复杂

## 依赖检查
- [ ] npm audit 无高危漏洞
- [ ] 没有不用的依赖
```

---

## 🔧 常用命令

```bash
# 检查漏洞
npm audit

# 自动修复
npm audit fix

# 检查过期依赖
npm outdated
```

---

## 🚨 如果被黑了

```
1. 改所有密码
2. 撤销所有 Token/Session
3. 检查有没有后门代码
4. 从备份恢复
5. 加固薄弱环节
```

---

## 📝 绝对禁止

- ❌ 明文存密码
- ❌ SQL 字符串拼接
- ❌ 把 secret 提交到 Git
- ❌ 用 HTTP 传密码
