# AGENTS.md — 协作与沟通

> 一个人开发，也需要和自己"协作"。

---

## 🎯 Lite版协作原则

虽然你是单人开发，但你需要和以下角色"协作"：

| 角色 | 说明 |
|------|------|
| 未来的你 | 代码要让半年后的你能看懂 |
| 用户 | 理解用户需求，快速响应 |
| AI助手（我） | 清晰描述需求，快速确认方案 |

---

## 📢 沟通原则

### 和用户/自己沟通

```
❌ "这个功能有点问题"
✅ "登录功能在密码错误时没有提示，需要修复"
```

### 和AI助手（我）沟通

```
❌ "帮我做个网站"
✅ "帮我做一个Todo应用，用Next.js + Prisma，需要用户登录"
```

**清晰的需求 = 更快的交付**

---

## 🔄 工作节奏

### 任务流程

```
需求 → 确认方案(5分钟) → 开发 → 自测 → 上线
```

### 遇到问题

```
1. 自己尝试解决（15分钟）
2. 搜Google/Stack Overflow
3. 问我
4. 如果卡住超过1小时，先跳过，做其他功能
```

---

## 📋 任务管理

### 简单Todo清单

```markdown
## 当前任务
- [ ] 功能A
- [ ] 功能B

## 进行中
- [ ] 功能C (50%)

## 已完成
- [x] 功能D ✅
```

### Git提交规范

```bash
feat: 新功能
fix: 修复bug
docs: 文档
chore: 杂项
```

---

## ⚡ 快速决策

### 不纠结

```
方案A vs 方案B？
→ 选简单的那个
→ 选熟悉的那个
→ 选能快速上线的那个
```

### 什么时候需要思考

- 架构选型（选完就定了，不好改）
- 数据库设计（改起来麻烦）
- 安全相关（底线问题）

---

## 🚫 避免

- ❌ 过度讨论，不行动
- ❌ 一个人憋太久不求助
- ❌ 完美主义，迟迟不上线
