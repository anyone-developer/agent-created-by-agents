# workflows/bug-fix.md — Bug修复流程

> 每个Bug都是一次学习机会。

---

## 🐛 Bug修复全流程

```
Bug报告 → 复现确认 → 问题定位 → 原因分析 → 修复方案 → 代码修复 → 验证测试 → 上线部署 → 复盘总结
```

---

### 严重程度定义

| 级别 | 定义 | 示例 | 响应时间 |
|------|------|------|---------|
| **致命** | 系统崩溃、数据丢失、核心功能完全不可用 | 无法登录、支付失败 | 1小时内 |
| **严重** | 核心功能异常、影响大部分用户 | 搜索无法使用、订单无法创建 | 4小时内 |
| **一般** | 非核心功能异常、有临时解决方案 | 样式错乱、提示信息错误 | 24小时内 |
| **轻微** | 体验问题、不影响功能 | 按钮颜色不对、文案错误 | 一周内 |

## 📋 Phase 2: 问题定位

### 定位策略

```markdown
## 问题定位

### 1. 信息收集
- 错误日志在哪里？
- 最近有什么改动？（git log）
- 什么时候开始出现的？
- 影响哪些用户？

### 2. 缩小范围
- 是前端问题还是后端问题？
- 是数据问题还是逻辑问题？
- 是环境问题还是代码问题？
- 是必现还是偶现？

### 3. 复现环境
- 能在本地复现吗？
- 能在测试环境复现吗？
- 需要什么特殊数据/条件？

### 4. 日志分析
```

### 常见Bug类型及定位方法

```markdown
## Bug类型诊断

### 前端Bug
**症状：** 页面异常、交互无响应、样式错乱
**定位方法：**
1. 打开浏览器开发者工具
2. 检查Console错误
3. 检查Network请求
4. 检查Elements元素
5. 使用React/Vue DevTools

### 后端Bug
**症状：** API报错、数据异常、服务不可用
**定位方法：**
1. 检查服务器日志
2. 检查数据库状态
3. 使用API测试工具复现
4. 检查最近的代码改动

### 数据库Bug
**症状：** 数据不一致、查询超时、数据丢失
**定位方法：**
1. 检查慢查询日志
2. 分析执行计划
3. 检查数据完整性
4. 检查锁和死锁

### 环境Bug
**症状：** 特定环境异常、配置问题
**定位方法：**
1. 对比环境配置
2. 检查环境变量
3. 检查依赖版本
4. 检查网络连接
```

### 修复原则

```markdown
## 修复原则

### 最小改动原则
- 只改必要的代码
- 不借机重构（除非风险很低）
- 保持改动范围可控

### 安全性原则
- 考虑修复的副作用
- 不引入新问题
- 保留回滚能力

### 完整性原则
- 修复问题本身
- 修复类似问题
- 添加测试防止回归
```

### 紧急修复（Hotfix）流程

```markdown
## Hotfix流程

### 适用场景
- 生产环境致命Bug
- 影响大量用户
- 需要立即修复

### 流程
1. 从main分支创建hotfix分支
   ```bash
   git checkout main
   git pull
   git checkout -b hotfix/fix-xxx
   ```

2. 最小改动修复

3. 本地测试通过

4. 提交PR，快速Review

5. 合并到main，立即部署

6. Cherry-pick到develop分支
   ```bash
   git checkout develop
   git cherry-pick <commit-hash>
   ```
```

- [ ] 添加了单元测试

- [ ] 测试了边界情况
```

### 修复示例

```typescript
// ❌ 不好的修复：只处理症状
function divide(a: number, b: number) {
  return a / b;  // 如果b=0会出问题
}

// ✅ 好的修复：处理根本问题
function divide(a: number, b: number): number {
  if (b === 0) {
    throw new Error('除数不能为0');
  }
  return a / b;
}

// ✅ 更好的修复：处理根本问题 + 测试
function divide(a: number, b: number): number {
  if (b === 0) {
    throw new Error('除数不能为0');
  }
  return a / b;
}

// 测试
describe('divide', () => {
  it('应该正确计算除法', () => {
    expect(divide(10, 2)).toBe(5);
  });
  
  it('应该在除数为0时抛出错误', () => {
    expect(() => divide(10, 0)).toThrow('除数不能为0');
  });
});
```
