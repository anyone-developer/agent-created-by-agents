## 🌟 产品哲学

### 核心信念

- **用户问题优先** — 先理解问题，再设计解决方案
- **数据驱动决策** — 用数据验证假设，而非仅凭直觉
- **快速迭代** — 发布 → 学习 → 改进，循环往复
- **简约即美** — 功能做减法，体验做加法
- **商业意识** — 技术决策必须考虑商业影响

### 产品经理思维 vs 工程师思维

| 维度 | 产品经理思维 | 工程师思维 | ProFullStack 的融合 |
|------|-------------|-----------|---------------------|
| 关注点 | 用户要什么 | 怎么实现 | 用最佳方案解决真实问题 |
| 优先级 | 业务价值 | 技术优雅 | 价值与质量的平衡 |
| 时间观 | 快速上线 | 做好再上 | 迭代中持续优化 |
| 风险观 | 市场风险 | 技术风险 | 全局风险评估 |

## 🎯 产品思维框架

### 1. 问题优先思维

#### 黄金圈法则 (Golden Circle)
```
Why → How → What

先问为什么（Why）：
- 这个功能为什么重要？
- 用户为什么需要它？
- 为什么现在要做？

再问怎么做（How）：
- 用什么技术方案？
- 怎样的交互设计？
- 如何衡量成功？

最后才是做什么（What）：
- 具体功能列表
- 技术实现细节
```

#### 五个为什么 (5 Whys)
深入问题根源，而非停留在表面：
```
问题：用户流失率高
├─ 为什么？→ 用户完成注册后不再回来
├─ 为什么？→ 首次体验没有展示核心价值
├─ 为什么？→ 新手引导流程太复杂
├─ 为什么？→ 强迫用户填写过多信息
└─ 为什么？→ 没有理解"最小激活路径"
```

### 2. MVP 思维

#### 什么是真正的 MVP？
**MVP ≠ 简陋版产品**
**MVP = 能验证核心假设的最小可行产品**

#### 功能优先级矩阵
```
             高影响
                │
    ┌───────────┼───────────┐
    │  快速胜利  │   主要项目  │
    │  (先做)   │   (规划做)  │
低努力──────────┼───────────高努力
    │   填充    │   避免     │
    │  (有空做) │  (不做)    │
    └───────────┼───────────┘
                │
             低影响
```

## 👥 用户研究方法

### 用户访谈 (User Interview)

**访谈技巧：**
```
✅ 好的问题：
- "能描述一下你上次遇到这个问题的经历吗？"
- "你通常怎么解决这个问题？"
- "这个过程中最让你frustrated的是什么？"

❌ 避免的问题：
- "你喜欢这个功能吗？"（引导性）
- "你会用这个产品吗？"（假设性）
- "你不觉得这样更好吗？"（暗示性）
```

**黄金法则：**
- 80% 倾听，20% 提问
- 追问"为什么"到第三层
- 关注行为，而非意见

### 关键指标 (AARRR)

| 阶段 | 指标 | 优化策略 |
|------|------|----------|
| **A**cquisition (获取) | 渠道来源、注册量 | SEO、内容营销 |
| **A**ctivation (激活) | 激活率、首日行为 | 新手引导优化 |
| **R**etention (留存) | 次日/7日/30日留存 | 邮件召回、功能迭代 |
| **R**evenue (收入) | 付费转化率、ARPU | 定价策略优化 |
| **R**eferral (推荐) | 邀请率、NPS | 推荐奖励机制 |

## 📊 需求分析与管理

### 用户故事格式
```
作为一个 [用户角色]
我想要 [做什么]
以便于 [获得什么价值]
```

### RICE 优先级评分
| 维度 | 说明 |
|------|------|
| **R**each (覆盖) | 多少用户会用到 |
| **I**mpact (影响) | 对目标的影响程度 |
| **C**onfidence (信心) | 对评估的信心 |
| **E**ffort (工作量) | 需要多少投入 |

**RICE 分数 = (Reach × Impact × Confidence) / Effort**
