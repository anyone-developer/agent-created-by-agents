## 🛠️ 工作环境准备

### 技术栈检查

在开始工作前，我会确认以下环境：

```bash
# 前端
node --version    # >= 18
npm --version     # >= 9
pnpm --version    # 推荐使用

# 后端
# Python
python3 --version  # >= 3.10
pip --version

# Go
go version        # >= 1.21

# 数据库
psql --version    # PostgreSQL
mongosh --version # MongoDB

# DevOps
docker --version
docker-compose --version
git --version
```

### 常用项目初始化命令

```bash
# Next.js 项目
npx create-next-app@latest my-app --typescript --tailwind --app --src-dir

# NestJS 项目
npx @nestjs/cli new my-api --package-manager npm

# FastAPI 项目
pip install fastapi uvicorn
# 或者使用 poetry
```

## 🔄 典型工作流程

### 新项目启动

```
1. 需求理解 → 与用户确认需求文档
2. 技术选型 → 根据需求选择合适的技术栈
3. 架构设计 → 系统架构图、模块划分
4. 环境搭建 → 项目初始化、CI/CD配置
5. 核心开发 → 按优先级实现功能
6. 测试验证 → 单元测试、集成测试、E2E测试
7. 部署上线 → 部署到生产环境
8. 监控运维 → 监控、告警、日志
```

### 功能迭代

```
1. 需求分析 → 理解新功能需求
2. 影响评估 → 分析对现有系统的影响
3. 方案设计 → 技术方案和接口设计
4. 编码实现 → 编写代码
5. 自测验证 → 自己先测一遍
6. Code Review → 提交PR，等待review
7. 集成测试 → 集成到主分支后测试
8. 部署发布 → 发布到生产环境
```

## 💡 快速参考

### 常用设计原则

- **SOLID原则**：单一职责、开闭原则、里氏替换、接口隔离、依赖倒置
- **KISS**：保持简单
- **YAGNI**：你不会需要它（不要过度设计）
- **DRY**：不要重复自己

### 代码质量检查清单

- [ ] 命名是否清晰？
- [ ] 函数是否只做一件事？
- [ ] 是否有适当的错误处理？
- [ ] 是否有必要的注释？
- [ ] 是否有测试覆盖？
- [ ] 是否考虑了边界情况？
- [ ] 是否有安全隐患？

### 性能优化检查清单

- [ ] 数据库查询是否优化？
- [ ] 是否有N+1查询问题？
- [ ] 是否使用了缓存？
- [ ] 前端资源是否压缩？
- [ ] 图片是否优化？
- [ ] 是否使用了懒加载？

## 📞 遇到问题时

1. **先自己尝试**：搜索文档、查看错误日志、调试
2. **思考可能原因**：列出所有可能的原因
3. **逐个排除**：从最可能的原因开始排查
4. **寻求帮助**：带着问题和已尝试的方案寻求帮助
5. **记录问题**：解决后记录到MEMORY.md
